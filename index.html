Recommended Permanent Home

Keep snapshots separate from source code

This repository is the durable home for ZeroClaw metrics. The collector runs from scripts/build_dashboard.py, writes immutable snapshots under data/snapshots/, refreshes data/latest.json, derives data/daily.json and data/metrics.sqlite, and publishes this static index.html through GitHub Pages. The main application repository should link here rather than storing metrics snapshots in product source.

Collector: scripts/metrics Snapshots: separate metrics store SQLite: data/metrics.sqlite Dashboard: GitHub Pages /metrics Docs link: maintainer docs

Methodology

Investor-readable metric definitions and caveats

Metric familyMeaningWindowMethod

GitHub views/clonesAttention and source-install proxyRolling 14dGitHub Insights traffic API; no lifetime clone/view total is exposed.

Referrers and pathsDiscovery channelsRolling top 10GitHub Insights top referrers and popular content for the recent traffic window.

Release payload assetsBinary distributionCumulativeGitHub release asset counters, excluding install.sh bootstrap downloads.
Homebrew installsPackage-manager adoptionRolling 30d/90d/365dHomebrew anonymous install analytics, not lifetime downloads.

This dashboard follows a snapshot-first model aligned with CHAOSS-style project health reporting: raw platform responses are preserved under data/snapshots/, and aggregate JSON, SQLite, and HTML views are regenerated from those snapshots.
